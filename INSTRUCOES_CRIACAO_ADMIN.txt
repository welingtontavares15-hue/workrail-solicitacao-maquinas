⚠️  PROBLEMA ANTERIOR (AGORA CORRIGIDO)

ANTES DA CORREÇÃO:
  ❌ Qualquer pessoa que fazia login era automaticamente criada como "vendas"
  ❌ Não havia proteção de segurança
  ❌ Impossível ter super_admin sem ir ao Firebase Console

DEPOIS DA CORREÇÃO:
  ✅ Apenas usuários pré-criados podem fazer login
  ✅ Admin deve ser criado PRIMEIRO antes de outros usuários
  ✅ Interface segura com função especial para criar PRIMEIRO admin
  ✅ Sem auto-criação de usuários

================================================================================
🔧 COMO CRIAR PRIMEIRO ADMINISTRADOR
================================================================================

OPÇÃO 1: Via Painel Admin (RECOMENDADO)
───────────────────────────────────────

1. Abra workrail_v2.html no navegador
2. Login screen aparece (deixe em branco, não faz login ainda)
3. Abra o Console do Navegador:
   - Windows: F12 ou Ctrl + Shift + I
   - Mac: Cmd + Option + I
4. Na aba "Console", digite:
   criarPrimeiroAdmin()
5. Preencha:
   - E-mail: seu@email.com
   - Senha: Agosto@2026 (ou outra segura, mín. 6 caracteres)
   - Nome: Seu Nome Completo
6. Confirme no popup
7. Sucesso! Você recebe o UID

TESTE:
   - Faça logout (se necessário)
   - Login com seu@email.com + senha
   - Você entrará como "Administrador do Sistema"


OPÇÃO 2: Via Firebase Console (Se falhar a Opção 1)
───────────────────────────────────────────────────

1. Abra Firebase Console → Seu projeto
2. Authentication → Usuários → Adicionar usuário
   - E-mail: seu@email.com
   - Senha: Agosto@2026 (crie uma)
3. Copie o UID gerado (ex: abc123def456...)
4. Firestore → Collection 'usuarios' → Add document
   - ID do documento: [cole o UID aqui]
   - Campos:
     uid: seu@email.com
     nome: "Seu Nome Completo"
     email: seu@email.com
     perfil: "super_admin"
     fornecedor: null
     ativo: true
5. Save
6. Teste o login

================================================================================
📋 PERFIS DISPONÍVEIS
================================================================================

Ao criar usuários no painel Admin, escolha um destes perfis:

1. super_admin (Administrador do Sistema)
   - Acesso total
   - Gerencia usuários, fornecedores, modelos
   - Apenas 1-2 pessoas devem ter este

2. gestor (Gestor de Vendas)
   - Aprova ou rejeita solicitações
   - Anexa checklist de instalação
   - ~1-2 pessoas

3. adm (ADM de Vendas)
   - Revisa e encaminha ao fornecedor
   - Acompanha a entrega
   - ~1-2 pessoas

4. vendas (Representante de Vendas)
   - Abre solicitações
   - Acompanha o fluxo
   - ~3-5 pessoas

5. fornecedor_ebst (Fornecedor EBST)
   - Vê apenas pedidos EBST
   - Anexa Nota Fiscal
   - Preparação de máquinas

6. fornecedor_hobart (Fornecedor Hobart)
   - Vê apenas pedidos Hobart
   - Anexa Nota Fiscal
   - Preparação de máquinas

================================================================================
⚠️  CORRIGINDO USUÁRIOS CRIADOS COMO "VENDAS"
================================================================================

Se vocêjá criou usuários com login e eles aparecerão errados como "vendas":

PASSO 1: Editar no painel Admin
───────────────────────────────
1. Faça login como administrador (com a conta criada acima)
2. Vá para Configurações → Admin Sistema → Usuários
3. Clique no usuário que precisa corrigir
4. Mude o perfil de "Vendas" para o correto
5. Salve
6. ✅ Usuário pode fazer login novamente com perfil correto

PASSO 2: Alternativa (via Firebase Console)
────────────────────────────────────────────
1. Firestore → Collection 'usuarios' → encontre o documento do usuário
2. Clique para editar
3. Mude o campo 'perfil' de "vendas" para o valor correto
4. Save
5. Usuário faz login e carrega o perfil correto

================================================================================
❓ PERGUNTAS FREQUENTES
================================================================================

P: Esqueci a senha de admin?
R: Use Firebase Console → Authentication → Reset password
   Ou clique "Problemas de acesso?" na tela de login

P: Um usuário não consegue fazer login?
R: Verifique se:
   - Usuário foi criado no painel Admin (ou Firebase Console)
   - Campo 'ativo' está true no Firestore
   - E-mail e senha estão corretos
   - Se criado no Console, precisa de um documento na collection 'usuarios'

P: Usuário fez login mas aparece como "Vendas" quando deveria ser Gestor?
R: O Firestore ainda tem 'perfil: vendas'
   Solução: Edite no painel Admin ou Firebase Console

P: Onde está meu usuário na tabela de Admin?
R: Se criou via criarPrimeiroAdmin(), aparece automaticamente
   Se criou via Firebase Console, só aparece após fazer login uma vez

P: Posso ter múltiplos super_admin?
R: Pode, mas não é recomendado. Use 1-2 máximo.
   Outros devem ser "adm" ou "gestor"
