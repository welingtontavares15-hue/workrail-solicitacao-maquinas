PROBLEMA IDENTIFICADO
─────────────────────
Usuários com perfil 'super_admin' no Firestore entravam como 'VENDAS'.

ROOT CAUSE
──────────
Função carregarPerfilUsuario() tinha catch() genérico que FORÇAVA perfil 
para 'vendas' se qualquer erro ocorresse durante carregamento/migração.

MUDANÇAS APLICADAS
──────────────────

1. carregarPerfilUsuario() (linhas 1978-2088)
   ✓ Adicionado logging detalhado em cada etapa
   ✓ Adicionada validação contra lista de perfis válidos
   ✓ Migração de documento agora é NÃO-BLOQUEANTE
   ✓ Removido fallback automático para 'vendas' no catch

2. Tratamento de Erro em catch (linhas 2072-2088)
   ❌ ANTES: força perfil: 'vendas' silenciosamente
   ✓ DEPOIS: 
     - Faz logout do usuário
     - Mostra mensagem de erro ESPECÍFICA
     - Permite diagnóstico no console

3. Firestore Rules (linhas 4351-4358)
   ✓ Documentação atualizada para incluir 'super_admin'
   ✓ isInterno() agora: ['super_admin','vendas','gestor','adm']
   ✓ /usuarios agora permite read/write para super_admin

VERIFICAÇÕES
────────────
✓ Menu (aplicarControleMenu) — JÁ ESTAVA CORRETO
  - Inclui super_admin em todos os itens relevantes
